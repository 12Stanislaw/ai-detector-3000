# AI Detector 3000

AI Detector 3000 is an intelligent, full-stack text verification system designed to determine whether a given English document was authored by a human or automatically generated by advanced Large Language Models (LLMs) such as ChatGPT or Claude. 

The project features a lightweight graphical user interface (GUI) built with `tkinter` and implements a hybrid machine learning classification pipeline.

## 🚀 Project Overview

Classical text filters and statistical models are highly vulnerable to **Topic Bias** (e.g., memorizing specific words like *car* or *ecology* from the training set and failing when tested on topics like *love* or *philosophy*). 

AI Detector 3000 solves this problem by using a **Fused Features Mode** (Hybrid Approach) that decouples semantic content from purely stylistic and syntactic structures.

### Key Architectural Highlights:
* **Vocabulary Analysis:** A heavily regularized `TfidfVectorizer` capped at 500 features with bi-grams to capture text semantics without overfitting to topics.
* **Feature Scaling:** Integration of `StandardScaler` to normalize linguistic metrics, ensuring micro-scale density features are not drowned out by large sentence length variances.
* **Stas & Bohdan Linguistic Metrics:** An advanced custom mathematical engine extracting 10 stylistic features, including:
  * *Sentence Length Variance* (Human text shows a jagged rhythm with high variance ~192, while AI generates rigid, metronome-like text with low variance ~19).
  * *Lexical Richness (Type-Token Ratio - TTR)*.
  * *Part-of-Speech (POS) Tagging* (e.g., Adjective and Stopword Density).
* **Balanced Dataset:** Trained on a meticulously balanced (50/50), multi-domain dataset compiled from localized academic essays and Kaggle's *DAIGT-V4* open-domain corpus, resulting in a cross-validation accuracy of **~91%**.

## 🛠️ Installation Guide

Follow these steps to set up and run the AI Detector 3000 environment on your local machine (tested on Fedora Linux).

### Prerequisites
Make sure you have Python 3.10+ installed. You will also need `pip` and python-tkinter binaries if they are not bundled with your Linux distribution.

### 1. Clone the Repository
For Fedora users:
```bash
sudo dnf install python3-tkinter
git clone [https://github.com/12Stanislaw/ai-detector-3000.git]
cd ai-detector-3000
```

### 2. Create and Activate a Virtual Environment
```bash
python3 -m venv venv
source venv/bin/activate
```

### 3. Install Dependencies
Install all required machine learning and natural language processing libraries:
```bash
pip install -r requirements.txt
```

### 4. Download NLTK Models

Since the project relies on NLTK for tokenization and POS tagging, you must download the required language assets. Run the following command inside your activated virtual environment:

```bash
python -c "import nltk; nltk.download('punkt'); nltk.download('averaged_perceptron_tagger')"
```

## 💻 How to Run

To automatically load the hybrid dataset, train the Logistic Regression classifier, and open the graphical application, simply run:
```bash
python main.py
```
Input Text: Paste any English text into the interface window and hit "Analyze" to see the real-time AI probability breakdown
